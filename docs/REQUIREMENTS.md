# 要件定義書

**プロジェクト名:** WordPress Central MCP Server  
**作成日:** 2026-06-25  
**ステータス:** 確定

---

## 1. 背景・目的

### 1.1 背景

複数の WordPress サイトを運用するにあたり、各サイトに AI 操作機能（アビリティ）を提供するプラグインを個別に管理している。現状は機能追加・修正のたびに全サイトへのプラグイン更新作業が発生し、サイト数の増加とともに運用コストが増大している。

### 1.2 目的

アビリティ定義を中央サーバーに集約することで、複数 WordPress サイトの機能管理を一元化し、運用コストを削減する。

### 1.3 解決する課題

| 課題 | 現状 | 解決後 |
|---|---|---|
| アビリティ更新 | 全サイトにプラグイン更新（N回） | 中央サーバーのみ更新（1回） |
| 新サイト追加 | プラグイン複数インストール＋設定 | mcp-adapter 1つ＋接続情報登録 |
| 品質管理 | サイトごとにバラつく可能性 | 中央で一元管理 |

---

## 2. スコープ

### 対象

- WordPress サイトの初期構築支援
- WordPress サイトの機能追加・改修支援
- 複数サイトの横断的な管理操作

### 対象外

- WordPress サイトのコンテンツ更新（記事・ページの作成・編集はWP管理画面で完結）
- エンドユーザー向けのリアルタイム処理
- WordPress 以外の CMS・サービス

---

## 3. ステークホルダー

| 役割 | 利用シーン |
|---|---|
| 開発者・エンジニア | 初期構築、機能追加、改修 |
| 運用担当者 | サイト設定変更、トラブル対応 |

---

## 4. 機能要件

### 4.1 サイト管理

| # | 要件 |
|---|---|
| F-01 | 複数の WordPress サイトを登録・管理できること |
| F-02 | 登録済みサイトの一覧を取得できること |
| F-03 | 指定したサイトで使用可能なアビリティ一覧を取得できること |

### 4.2 アビリティ管理

| # | 要件 |
|---|---|
| F-04 | アビリティ定義を中央サーバーで一元管理できること |
| F-05 | 指定したサイトで任意のアビリティを実行できること |
| F-06 | アビリティの追加・修正は中央サーバーの再デプロイ1回で、全サイトへの操作に適用されること |

### 4.3 認証・セキュリティ

| # | 要件 |
|---|---|
| F-07 | Claude から中央サーバーへの接続を API キーで保護すること |
| F-08 | 各 WP サイトへの接続情報を Secret Manager で安全に管理すること |

---

## 5. 非機能要件

### 5.1 可用性
- コールドスタートを許容する（開発ツール用途のため数秒の遅延は問題なし）

### 5.2 性能
- 同時接続数: 少数（開発者・運用者のみが利用）
- レスポンス: 数秒以内（厳密な SLA は不要）

### 5.3 拡張性
- サイト数が増えてもアビリティの管理コストが増加しないこと

### 5.4 コスト
- 無料枠内で運用できること（低トラフィック用途）

---

## 6. 制約条件

- 実行環境: Google Cloud Run（プロジェクト: `wordpressmcp` / ID: 621428460182）
- リージョン: asia-northeast1
- 各 WP サイトは `mcp-adapter` プラグインが導入済みであること
- MCP プロトコル（Streamable HTTP）に準拠すること

---

## 7. 前提条件

- コンテンツ更新は引き続き WP 管理画面で行う（MCP の対象外）
- MCP を利用するのは開発者・運用者のみ
- 利用頻度は初期構築・機能追加・改修時のみであり、日常的なトラフィックはほぼ発生しない

---

## 8. 用語定義

| 用語 | 定義 |
|---|---|
| アビリティ | WP サイトに対して実行できる操作の単位（例: 投稿作成、投稿削除） |
| mcp-adapter | 各 WP サイトに導入する薄いブリッジプラグイン。WP 関数の実行のみを担う |
| enable-abilities-for-mcp | 現状各サイトにアビリティ定義を持つプラグイン。中央サーバー移行後は廃止 |
| 中央 MCP サーバー | アビリティ定義を持ち、複数サイトへの操作を仲介するサーバー |
| MCP | Model Context Protocol。Claude と外部システムを繋ぐプロトコル |
